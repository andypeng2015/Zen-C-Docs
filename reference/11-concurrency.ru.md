# 11. Параллельность (Async/Await)

Zen C использует модель **бесстековых корутин** для async/await -- без пула потоков, без зависимости от pthread.
Это означает, что асинхронные функции преобразуются во время компиляции в конечные автоматы, а не в потоки.

#### Синтаксис

Объявите асинхронную функцию с помощью `async fn` и ожидайте её результат с помощью ключевого слова `await`:

```zc
async fn fetch_data() -> string {
    return "Данные";
}

fn main() {
    let result = await fetch_data();
}
```

{% alert(type="note") %}
В отличие от моделей на основе потоков, `await` в Zen C -- это **блокирующий цикл опроса**, который выполняет
всю асинхронную функцию до завершения. Это сделано намеренно -- делает поток управления
предсказуемым и устраняет необходимость в исполнителе или пуле потоков.
{% end %}

#### Как это работает

`async fn` преобразуется компилятором в **конечный автомат**. Каждая
точка `await` становится переходом состояния. Результирующая структура `Future`
содержит состояние, параметры и под-футуры для ожидаемых вызовов.

#### Последовательный шаблон

```zc
async fn task(n: int) -> int {
    return n * 2;
}

fn main() {
    let a = await task(5);
    let b = await task(a);
}
```
