# 13. Comptime (Выполнение на этапе компиляции)

Comptime позволяет выполнять код Zen на этапе компиляции для генерации исходного кода, проверки констант и вывода диагностики.

```zc
comptime {
    yield("def build_date = \"2024-01-01\";");
}

println "Build Date: {build_date}";
```

## 12.1 Вспомогательные функции

| Функция | Описание |
|:---|---|
| `yield(str)` | Вывод сгенерированного кода |
| `code(str)` | Псевдоним для `yield()` |
| `compile_error(msg)` | Остановить компиляцию с ошибкой |
| `compile_warn(msg)` | Вывести предупреждение на этапе компиляции |

## 12.2 Переменные и управляющие конструкции

```zc
comptime {
    let sum = 0;
    for i in 0..5 { sum = sum + i; }
    expect(sum == 10, "цикл работает");
}
```

## 12.3 Проверки в comptime

`expect()` проверяет условия на этапе компиляции:

```zc
comptime {
    expect(1 + 1 == 2, "математика OK");
}
```

## 12.4 Метаданные сборки

| Константа | Тип | Описание |
|:---|---:|---|
| `__COMPTIME_TARGET__` | string | Платформа: `"linux"`, `"windows"`, `"macos"` |
| `__COMPTIME_FILE__` | string | Текущий исходный файл |

## 12.5 Функции `@comptime`

Функции, помеченные `@comptime`, могут использоваться как в comptime-блоках, так и во время выполнения.

## 12.6 Советы

Используйте сырые строки (`r"..."`) в comptime, чтобы избежать проблем с фигурными скобками: `code(r"fn test() { return 42; }")`.
